# 深入 Claude Code 源码：AI Agent 应用的全栈技术解析

> **从一份真实的 AI 产品源码中，学会构建 AI Agent 应用的全栈技术**

## 关于本书

本书是基于 Anthropic **Claude Code** 完整源码的深度技术分析。Claude Code 是一个 AI 驱动的命令行编程助手，源码涵盖了 System Prompt 工程、多 Agent 编排、工具系统、终端 UI、Bridge IPC、远程会话、企业代理等完整技术栈。

我们将以「源码阅读」的方式逐模块解析这个真实的生产级 AI 产品，整书以**运行期生命周期**为主线——进入软件之前 → 一次回合的内核 → 工具家族 → Agent/任务/协调 → 协议安全与扩展 → 网络与远程 → 终端 UI 与多模态输入 → 记忆/扩展/总结——从中提炼出可迁移到你自己项目的设计模式与工程实践。

> 📝 想了解本书的修订历程，见根目录下的 [修订说明](../V2-CHANGELOG.md)。

### 目标读者

- 有 1–3 年经验的程序员，了解 TypeScript
- 对 AI 应用开发感兴趣，想从真实产品中学习
- 希望理解大型 AI CLI 产品的架构设计

### 阅读约定

- 正文使用中文，技术术语保留英文原文（如 Agent、Tool、System Prompt）
- 代码引用格式：`文件路径:行号范围`
- 每章可独立阅读，但建议按顺序——章节顺序就是一次会话从启动到产出的链路本身

## 阅读建议

1. **入门路线**（7 章）：第 1 → 2 → 33 → 5 → 10 → 14 → 34，从全景到启动到状态，再沿 query → tools → agent → 总结的主干建立全局认知。
2. **AI 工程路线**（9 章）：第 1 → 33 → 6 → 5 → 7 → 9 → 10 → 14 → 15，先建立运行上下文，再深入 prompt / query / compact / thinking / agent 的 AI 核心设计（第 8 章 Prompt Cache 是横切主题，贯穿 6/5/7 章的缓存策略，建议在读完主干后再看）。
3. **远程与企业路线**（5 章）：第 3 → 4 → 23 → 24 → 25，从配置合并、迁移代码到客户端传输、Bridge IPC、DirectConnect/上游代理，看一个 CLI 如何在企业拓扑里活下来。
4. **完整路线**（34 章）：按顺序阅读，获得最完整的理解。
